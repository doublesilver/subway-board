# 출퇴근길 익명 채팅 - 기획 명세서

## 프로젝트 개요

### 서비스명
**가기싫어** (gagisiro.com)

### 컨셉
서울 지하철 호선별로 나뉜 익명 채팅방에서 같은 호선을 이용하는 통근러들이 자유롭게 소통하는 플랫폼

### 핵심 가치
- **호선 기반 커뮤니티**: 같은 노선을 이용하는 사람들끼리의 공감대
- **완전한 익명성**: 부담 없는 자유로운 소통
- **실시간 채팅**: 메신저 형태의 즉각적인 대화
- **안전한 환경**: 매일 자정 자동 삭제되는 메시지

---

## 타겟 사용자

### 주 타겟
- 서울 지하철 통근/통학자
- 20-40대 직장인
- 출퇴근 시간 여유 있는 사용자

### 사용 시나리오
1. **출근길 지하철 내**: "오늘 회사 가기 싫다"
2. **퇴근길 정류장**: "오늘 야근 언제 끝나려나"
3. **점심시간**: "이번 역 근처 맛집 추천해주세요"

---

## 주요 기능

### 1. 익명 로그인
- **클릭 한 번으로 즉시 입장**
- 자동 생성 닉네임 (예: "활기찬 통근러123")
- 회원가입/개인정보 수집 없음

### 2. 호선 선택
- 1호선 ~ 9호선 (9개 채널)
- 호선별 실시간 접속자 수 표시
- 서울 지하철 공식 노선 색상 적용

### 3. 실시간 채팅
- Socket.IO 기반 실시간 메시지
- 답장(Reply) 기능
- 입장/퇴장 시스템 메시지
- 메시지 길이: 1-1000자

### 4. 운영 정책
- **운영 시간**: 평일 07:00 ~ 09:00 (베타 기간 24시간)
- **메시지 초기화**: 매일 자정(00:00)
- **비속어 필터링**: 자동 필터링 적용

### 5. 피드백 시스템
- 서비스 개선을 위한 사용자 피드백 수집
- 익명 제출 가능

## 보안 및 개인정보

### 개인정보 처리
- **수집하지 않는 정보**: 이름, 이메일, 전화번호
- **최소 수집**: 익명 세션 ID (UUID)

### 데이터 보관
- **메시지**: 매일 자정 자동 삭제
- **세션**: 브라우저 세션 종료 시 삭제

### 보안 조치
- HTTPS 통신
- Rate Limiting (쓰기: 15분당 50회)
- XSS/SQL Injection 방지
- 비속어 필터링
